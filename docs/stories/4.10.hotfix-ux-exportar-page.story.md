# Story 4.10: Hotfix UX — ExportarPage: Stepper, Preview, Voltar e Fidelidade

## Status

Ready

## Story

**As a** operador da ferramenta,
**I want** que a tela Exportar mostre o stepper completo, permita abrir o preview, voltar à geração para ajustes e exiba o score de fidelidade final,
**so that** eu tenha contexto visual do progresso concluído, possa verificar o resultado e fazer últimos ajustes sem perder o trabalho.

## Acceptance Criteria

1. `ExportarPage.vue` usa `WizardLayout` (com stepper) em vez de `FullWidthLayout` — step 5 ativo, todos os steps anteriores marcados como completos
2. Botão `[🔍 Abrir Preview]` presente na ExportarPage — ao clicar, abre `${API_BASE}/api/preview/${generation.previewJobId}` em nova aba (`window.open(..., '_blank')`)
3. Botão `[◀ Voltar à Geração]` presente — navega para `/geracao` via `router.push('/geracao')`
4. Score de fidelidade final exibido: `FidelityScore` atom + barra de progresso + `fidelityComment` — dados lidos de `useGenerationStore()`
5. Botão "Abrir Preview" fica desabilitado se `generation.previewJobId` for null
6. `tsc --noEmit` passa sem erros
7. `npm run build` conclui sem erros
8. Fluxo Upload → Campos → Layout → Geração → Exportar não regride

## Dev Notes

### Bug B-UX-E1 — FullWidthLayout vs WizardLayout

```vue
<!-- ATUAL (errado): -->
<FullWidthLayout>
  ...
</FullWidthLayout>

<!-- CORRETO: -->
<WizardLayout :current-step="5" :show-save="true" :show-bibliotecas="false">
  ...
</WizardLayout>
```

O wireframe (Tela 5) mostra o stepper com todos os 5 steps marcados (●●●●●). `FullWidthLayout` não inclui stepper. `WizardLayout` já passa `:currentStep` para o `AppHeader` que exibe o `WizardStepper`.

### Bug B-UX-E2 — Abrir Preview ausente

`ExportarPage` tem botão "Download ZIP" e "Salvar Projeto" mas não "Abrir Preview". O wireframe especifica `[🔍 Abrir Preview]`. A URL já é padrão do sistema: `/api/preview/{jobId}`. `generation.previewJobId` já está disponível no store.

### Bug B-UX-E3 — Voltar à Geração ausente

O wireframe especifica `[◀ Voltar à Geração]` para permitir ajustes de última hora. Implementação simples: `router.push('/geracao')`. Não é necessário aviso pois nenhum estado é perdido ao voltar.

### Bug B-UX-E4 — Score de fidelidade final ausente

O score calculado na GeracaoPage fica em `generationStore.fidelityScore`. A ExportarPage deve exibí-lo como confirmação do resultado final. Exemplo de layout:

```vue
<section v-if="generation.fidelityScore !== null" class="fidelity-final">
  <h2>🎯 Fidelidade Final</h2>
  <FidelityScore :score="generation.fidelityScore" />
  <ProgressBar :value="generation.fidelityScore" />
  <p v-if="generation.fidelityComment">{{ generation.fidelityComment }}</p>
</section>
```
