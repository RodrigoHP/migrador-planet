# Story 4.2: Endpoint /api/generate e Preview via GET

## Status

Ready

## Story

**As a** operador da ferramenta,
**I want** que a tela de Geração dispare a criação do template via backend e exiba o preview corretamente,
**so that** eu possa ver o HTML/CSS/JS gerado com o score de fidelidade antes de exportar.

## Acceptance Criteria

1. `POST /api/generate` aceita body `{ mappingFields: FieldMapping[], layoutConfig: object, monacoEdits: { html?: string, css?: string, js?: string } }` e retorna `{ "jobId": "<uuid>" }` imediatamente
2. Pipeline de geração executa em background via `asyncio.create_task()`, emitindo eventos SSE: `step` com "Gerando HTML" (pct 30), "Calculando fidelidade" (pct 70), `done` (pct 100)
3. Após receber evento `done` via SSE, o frontend busca o resultado via `GET /api/result/{jobId}` que retorna `{ html, css, js, exemplo, fidelityScore, fidelityComment, iaSuggestions }`
4. Frontend (`GeracaoPage.vue`) popula `generation.html`, `generation.css`, `generation.js`, `generation.fidelityScore`, `generation.fidelityComment`, `generation.iaSuggestions`
5. `GET /api/preview/{job_id}` retorna HTML renderizado com status 200 (substitui o `POST /api/preview` com body)
6. `window.open('/api/preview/${jobId}')` no frontend abre o preview em nova aba corretamente
7. Frontend (`GeracaoPage.vue`) chama `/api/generate` (com prefixo `/api/`) em vez de `/generate`
8. `monacoEdits` aplicados como override pós-geração: se `monacoEdits.html` existir, sobrescreve o `html` gerado no resultado retornado
9. Score de fidelidade exibido na UI: número 0–100 e comentário descritivo
10. Nenhuma chamada retorna 404 ou 405 (Method Not Allowed) no fluxo de geração

## Dev Notes

### Causa Raiz dos Gaps G6 e G8

**G6 (POST /generate não existe):**
O frontend chama `POST /generate` com `{ mappingFields, layoutConfig, monacoEdits }` esperando iniciar um job de geração. O backend não tinha esse endpoint — a geração de template era embutida no pipeline de extração. Esta story cria o endpoint separado para o fluxo de geração pós-edição.

**G8 (POST vs GET preview):**
O backend tinha `POST /api/preview` com body `{ jobId }`, mas o frontend faz `window.open(GET /preview/{id})` — o que é incompatível (browser não envia body em window.open). Solução: criar `GET /api/preview/{job_id}` com `job_id` no path.

### Novo endpoint POST /api/generate — Estrutura

```python
# backend/routers/generate.py
from fastapi import APIRouter
from pydantic import BaseModel
from typing import List, Optional
import asyncio, uuid

from models.field_mapping import FieldMapping
from services import job_manager
from services.template_generator import TemplateGenerator
from services.fidelity_scorer import FidelityScorer

router = APIRouter()

class MonacoEdits(BaseModel):
    html: Optional[str] = None
    css: Optional[str] = None
    js: Optional[str] = None

class GenerateRequest(BaseModel):
    mappingFields: List[dict]
    layoutConfig: dict = {}
    monacoEdits: MonacoEdits = MonacoEdits()

@router.post("/generate")
async def start_generate(request: GenerateRequest):
    job_id = str(uuid.uuid4())
    job_manager.create_job(job_id)
    asyncio.create_task(run_generation_pipeline(job_id, request))
    return {"jobId": job_id}

async def run_generation_pipeline(job_id: str, request: GenerateRequest):
    try:
        # Converter dicts em FieldMapping (Pydantic v2)
        fields = [FieldMapping.model_validate(f) for f in request.mappingFields]

        # Gerar template
        gen = TemplateGenerator()
        output = gen.generate(fields, blocks=[])
        await job_manager.emit(job_id, "step", {"step": "Gerando HTML", "pct": 30})

        # Score de fidelidade
        scorer = FidelityScorer()
        fidelity = scorer.score(fields)
        await job_manager.emit(job_id, "step", {"step": "Calculando fidelidade", "pct": 70})

        # Aplicar monacoEdits como override
        html = request.monacoEdits.html or output["html"]
        css = request.monacoEdits.css or output["css"]
        js = request.monacoEdits.js or output["js"]

        result = {
            "html": html,
            "css": css,
            "js": js,
            "exemplo": output["exemplo"],
            "fidelityScore": fidelity["score"],
            "fidelityComment": fidelity["comment"],
            "iaSuggestions": fidelity["suggestions"],
        }
        job_manager.set_result(job_id, result)
        await job_manager.emit(job_id, "done", {})
    except Exception as exc:
        job_manager.set_error(job_id, str(exc))
        await job_manager.emit(job_id, "error", {"message": str(exc)})
```

onDone: async () => {
  const jobId = generation.previewJobId

  if (!generation.previewJobId) return

### Nota sobre TemplateGenerator com blocks=[]

O `TemplateGenerator.generate(fields, blocks)` foi originalmente pensado para receber `TextBlock[]` do PDF para posicionamento absoluto. Quando chamado via `/api/generate` (pós-extração), `blocks` pode ser vazio — o gerador deve funcionar apenas com os campos mapeados, gerando CSS de posicionamento relativo. Verificar se `TemplateGenerator` trata `blocks=[]` graciosamente.
