# Story 1.4: Step 3 — Layout

## Status

ReadyForReview

## Story

**As a** Operador Técnico,
**I want** configurar o layout visual do template — margens, fontes, cores — e selecionar as versões das Bibliotecas a incluir no ZIP, com um preview que atualiza em tempo real,
**so that** o HTML gerado siga as especificações visuais corretas antes de iniciar a geração.

## Acceptance Criteria

1. Rota `/layout` renderiza `LayoutPage` com dois painéis: controles (40%) e preview (60%)
2. `LayoutControls` organism exibe campos: pageSize (select), marginTop/Bottom/Left/Right (number inputs em mm), baseFontFamily (select), baseFontSize (number), primaryColor (ColorPicker), lineHeight (number)
3. Preview atualiza em tempo real com debounce de 300ms ao modificar qualquer controle
4. Valores default pré-preenchidos: A4, 20mm margens, `Inter`, 14px, `#2563EB`, 1.5
5. Botão "Bibliotecas" no `AppHeader` abre `BibliotecasModal`
6. `BibliotecasModal` lista Bibliotecas disponíveis (via `GET /bibliotecas/catalog`), exibe versão ativa e permite troca; ao clicar "Aplicar" atualiza `layoutStore.bibliotecasVersions`
7. Botão "Confirmar Layout": `layoutStore.confirmed = true`, router navega para `/geracao`
8. `layoutStore` persiste seleções via `idb` (IndexedDB) para retomada de projeto
9. `ColorPicker` atom: `<input type="color">` + preview inline da cor selecionada
10. `FontSelector` atom: select com opções de fontes disponíveis (Inter, Roboto, Open Sans)

## Dev Notes

### Layout Store — Interface
[Source: docs/front-end-spec.md#9-stores-pinia-e-composables]
```typescript
interface LayoutStore {
  pageSize: 'A4' | 'Letter' | 'A3'
  marginTop: number; marginBottom: number; marginLeft: number; marginRight: number  // mm
  baseFontFamily: string
  baseFontSize: number  // px
  primaryColor: string  // hex
  lineHeight: number
  bibliotecasVersions: Record<string, string>  // { knockout: '3.5.1', ... }
  confirmed: boolean
}
```
