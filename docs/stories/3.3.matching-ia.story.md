# Story 3.3: Matching IA com Campos JSON/XSD

## Status

Ready for Review

## Story

**As a** dev agent,
**I want** implementar matching semântico entre blocos extraídos do PDF e campos do contrato (JSON/XSD),
**so that** o motor de migração consiga mapear automaticamente os campos do documento com os campos de dados estruturados.

## Acceptance Criteria

1. `XSDParser` extrai campos do XSD com: `name`, `type`, `min_occurs`, `max_occurs`, `description`
2. `DataParser` extrai campos e valores de arquivos JSON e XML
3. Matching por nome normalizado: sem acento, lowercase, remoção de caracteres especiais
4. Score de confiança por campo: 🟢 `confidence >= 0.8`, 🟡 `0.5 <= confidence < 0.8`, 🔴 `confidence < 0.5`
5. Normalização de formatos BR aplicada: moeda (R$ X.XXX,XX → float), datas (DD/MM/YYYY → ISO), CEP (XXXXX-XXX), telefone
6. Validação cruzada XSD vs dados: detectar campos presentes no XSD mas ausentes no JSON/XML e vice-versa
7. Retornar `List[FieldMapping]` com todos os campos e seus scores
8. Integrar na rota `POST /api/jobs` como fase 2: emitir SSE `step: "Matching de campos"`, `pct: 60`
9. `FieldMapping` alinhado com o contrato do frontend (`frontend/src/types/index.ts`)

### Normalização BR — Referência

```python
import unicodedata, re

def normalize_name(name: str) -> str:
    # Remove acentos
    nfkd = unicodedata.normalize("NFKD", name)
    ascii_str = nfkd.encode("ascii", "ignore").decode("ascii")
    # Lowercase, substituir separadores
    return re.sub(r"[-_\s]+", " ", ascii_str).strip().lower()

def normalize_br_currency(value: str) -> Optional[float]:
    # R$ 1.234,56 → 1234.56
    cleaned = re.sub(r"[R$\s.]", "", value).replace(",", ".")
    try:
        return float(cleaned)
    except ValueError:
        return None

def normalize_br_date(value: str) -> Optional[str]:
    # DD/MM/YYYY → YYYY-MM-DD
    m = re.match(r"(\d{2})/(\d{2})/(\d{4})", value)
    if m:
        return f"{m.group(3)}-{m.group(2)}-{m.group(1)}"
    return None
```

### Distância de String (sem dependências externas)

```python
def levenshtein_similarity(a: str, b: str) -> float:
    if a == b:
        return 1.0
    m, n = len(a), len(b)
    if m == 0 or n == 0:
        return 0.0
    dp = list(range(n + 1))
    for i in range(1, m + 1):
        prev = dp[:]
        dp[0] = i
        for j in range(1, n + 1):
            cost = 0 if a[i-1] == b[j-1] else 1
            dp[j] = min(dp[j-1]+1, prev[j]+1, prev[j-1]+cost)
    return 1 - dp[n] / max(m, n)
```
