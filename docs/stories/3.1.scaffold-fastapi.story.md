2. CORS configurado para origem `http://localhost:5173`
3. `POST /api/upload/pdf` aceita arquivo PDF multipart, **cria** um novo `jobId` (UUID), salva em `/tmp/jobs/{jobId}/input.pdf`, retorna `{ jobId: string }`

7. `uvicorn main:app --reload --port 8000` inicia servidor sem erros
8. `backend/.env.example` documenta variáveis de ambiente disponíveis
9. Frontend consegue fazer upload de PDF via UI sem erro de CORS ou 404

### Implementação de Referência

**main.py:**
```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from routers import upload

app = FastAPI(title="Migrador Planet API")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

app.include_router(upload.router, prefix="/api")

@app.get("/api/health")
async def health():
    return {"status": "ok"}
```

**routers/upload.py — pattern de jobId:**
```python
import uuid, os
from pathlib import Path
from fastapi import APIRouter, UploadFile, File

router = APIRouter()

def create_job_dir(job_id: str) -> Path:
    path = Path(f"/tmp/jobs/{job_id}")
    path.mkdir(parents=True, exist_ok=True)
    return path

@router.post("/upload/pdf")
async def upload_pdf(file: UploadFile = File(...)):
    job_id = str(uuid.uuid4())
    job_dir = create_job_dir(job_id)
    content = await file.read()
    (job_dir / "input.pdf").write_bytes(content)
    return {"jobId": job_id}
```

### Nota sobre o jobId

**Apenas o upload do PDF cria o `jobId`** (UUID gerado no backend). Os uploads subsequentes de XSD e data recebem o `jobId` do PDF como form field obrigatório. O frontend deve armazenar o `jobId` retornado pelo upload do PDF e enviá-lo nos uploads seguintes. A Story 3.4 usará o mesmo `jobId` para iniciar o job via `POST /api/jobs`.

**Exemplo de form multipart para XSD:**
```
POST /api/upload/xsd
Content-Type: multipart/form-data
Fields: file=<arquivo.xsd>, jobId=<uuid-do-pdf>
```
