# Story 1.6: Step 5 — Exportar + Projeto

## Status

ReadyForReview

## Story

**As a** Operador Técnico,
**I want** validar o ZIP gerado, fazer o download do pacote autocontido, salvar o projeto como .json para retomada futura e poder reabrir projetos anteriores pela Home,
**so that** o fluxo completo esteja encerrado com um ZIP funcional e a opção de retomar qualquer projeto em qualquer momento.

## Acceptance Criteria

1. Rota `/exportar` renderiza `ExportarPage` com `FullWidthLayout` centralizado
2. `ExportChecklist` organism executa automaticamente ao entrar na tela e exibe validações do ZIP (estrutura, assets, Bibliotecas bundladas)
3. Botão "Download ZIP": `GET /export/{generation.previewJobId}/zip` → blob → `URL.createObjectURL(blob)` → download automático; `URL.revokeObjectURL` após 60s; exibe tamanho do arquivo
4. Botão "Salvar Projeto (.json)": `useProject.save()` → `showSaveFilePicker({ suggestedName: 'projeto.json' })` → serializa todos os stores
5. Arquivo .json salvo contém: metadados, step atual, `session.extraction`, `mappingStore.fields`, `layoutStore.$state`, `generation.monacoEdits`, `generation.chartConfigs`
6. Botão "Novo Projeto" na tela Exportar: confirma com dialog e navega para `/` resetando todos os stores
7. Na `HomePage`: botão "Abrir Projeto (.json)" → `useProject.load()` → `showOpenFilePicker` → desserializa stores → `router.push(step_salvo)`
8. `useProject.load()` restaura corretamente o step onde o projeto parou (ex: parou em `/campos` → navega para `/campos`)
9. Checklist items: verde (ok), amarelo (warning com nota), vermelho (erro crítico — desabilita download)
10. ZIP baixado contém Bibliotecas bundladas com caminhos relativos internos (`./js/knockout.js`)

## Dev Notes

### API Endpoint ZIP
[Source: docs/front-end-spec.md#12-integracao-backend]
```
GET ${API_BASE}/export/{jobId}/zip  // retorna blob application/zip
```

### ZIP Autocontido
[Source: docs/architecture/architecture.md — NFR7]
O ZIP gerado pelo backend inclui Bibliotecas bundladas com caminhos relativos internos.
HTML usa `./js/knockout.js` (não `../Bibliotecas/`). Isso é responsabilidade do backend.
Frontend apenas faz download do blob.

### useProject — Estrutura do .json
[Source: docs/front-end-spec.md#12-next-steps]
```typescript
interface ProjectFile {
  version: '1.0'
  savedAt: string  // ISO 8601
  currentStep: 0 | 1 | 2 | 3 | 4 | 5
  session: Omit<SessionStore, 'bytes'>  // sem ArrayBuffer (não serializa)
  mapping: { fields: FieldMapping[], confirmed: boolean }
  layout: LayoutStore
  generation: { monacoEdits: GenerationStore['monacoEdits'], chartConfigs: GenerationStore['chartConfigs'] }
}
```
**NOTA:** `ArrayBuffer` dos arquivos NÃO é serializado no .json. Na retomada, o usuário precisará re-selecionar os arquivos se quiser re-processar.

[Source: docs/front-end-spec.md#9-stores-pinia-e-composables]
