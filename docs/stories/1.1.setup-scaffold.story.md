# Story 1.1: Setup & Scaffold

## Status

Draft

## Story

**As a** dev agent,
**I want** um scaffold completo do projeto frontend com estrutura Atomic Design, Vite configurado, Tailwind com tokens customizados, Vue Router com guards, 4 Pinia stores tipados e 4 composables base,
**so that** todas as stories subsequentes possam ser implementadas sobre uma base sólida, consistente e sem ambiguidade arquitetural.

## Acceptance Criteria

1. Projeto Vite + Vue 3 + TypeScript criado em `frontend/` com `npm run dev` funcional na porta 5173
2. Tailwind CSS configurado com paleta customizada completa (tokens de `docs/front-end-spec.md` seção 6.1)
3. Vue Router configurado com 6 rotas (`/`, `/upload`, `/campos`, `/layout`, `/geracao`, `/exportar`) e guards de navegação implementados
4. 4 Pinia stores criados com interfaces TypeScript exatas conforme `docs/front-end-spec.md` seção 9: `useSessionStore`, `useMappingStore`, `useLayoutStore`, `useGenerationStore`
5. 4 composables criados com assinaturas TypeScript: `useSSE`, `useFileSystem`, `useProject`, `useBibliotecas`
6. Estrutura de pastas Atomic Design criada: `atoms/`, `molecules/`, `organisms/`, `templates/`, `pages/`
7. `AppHeader` organism criado com logo, wizard step indicator placeholder e botão "Bibliotecas" placeholder
8. `AppToast` organism criado com suporte a variantes success/warning/error/neutral e auto-dismiss 4s
9. `vite.config.ts` configurado com proxy `/api` → `http://localhost:8000` para dev e manual chunks para Monaco, PDF.js e Chart.js
10. `npm run build` produz `frontend/dist/` sem erros TypeScript

## Dev Notes

### Stack Frontend
[Source: docs/front-end-spec.md#1-introducao]
- Vue 3 + TypeScript + Vite
- Pinia 2.x + Vue Router 4.x
- Tailwind CSS (base unit 4px)
- lucide-vue-next (ícones)
- Inter (fonte UI), JetBrains Mono (código)

### Estrutura de Pastas
[Source: docs/front-end-spec.md#12-next-steps]
```
frontend/src/
  atoms/          # Button, Badge, ProgressBar, Spinner, Icon, Input, Select
  molecules/      # FileDropzone, FieldStatusBadge, CrossValidationBadge...
  organisms/      # WizardStepper, PDFViewer, FieldMappingTable, MonacoTabs...
  templates/      # WizardLayout, SplitPaneLayout, FullWidthLayout
  pages/          # HomePage, UploadPage, CamposPage, LayoutPage, GeracaoPage, ExportarPage
  stores/         # session.ts, mapping.ts, layout.ts, generation.ts
  composables/    # useSSE.ts, useFileSystem.ts, useProject.ts, useBibliotecas.ts
  router/         # index.ts com guards de navegação
  types/          # interfaces exportadas
  assets/         # main.css, fontes
```

### Guards de Navegação
[Source: docs/front-end-spec.md#2-arquitetura-de-informacao]
```typescript
// Regras de guard (em router/index.ts):
'/campos'   → requer: sessionStore.extraction !== null
'/layout'   → requer: mappingStore.confirmed === true
'/geracao'  → requer: layoutStore.confirmed === true
'/exportar' → requer: generationStore.html !== null
```

### Vite Config — Manual Chunks
[Source: docs/front-end-spec.md#11-performance]
Bundle target: < 400KB gzip chunk inicial. Monaco, PDF.js e Chart.js devem ficar em chunks separados.

### Testing

`tsc --noEmit` deve passar sem erros.
