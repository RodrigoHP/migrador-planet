# Story 1.2: Step 1 — Upload

## Status

Draft

## Story

**As a** Operador Técnico,
**I want** uma tela de upload onde posso selecionar PDF, XSD e arquivo de dados, ver a cross-validation automática e acompanhar o progresso da análise via SSE,
**so that** meus arquivos sejam enviados ao backend e eu receba o resultado da extração de campos sem precisar sair da tela.

## Acceptance Criteria

1. Rota `/upload` renderiza `UploadPage` com `WizardLayout` e `WizardStepper` indicando step 1 ativo
2. Três `FileDropzone` independentes para PDF, XSD e Dados (JSON/XML) usando `showOpenFilePicker()` da File System Access API — fallback para `<input type="file">` quando API não disponível
3. Após seleção dos 3 arquivos, `CrossValidationBadge` aparece com status `ok` ou `divergence` com lista de divergências
4. Botão "Analisar" habilitado somente após 3 arquivos selecionados; desabilitado durante processamento
5. Ao clicar "Analisar": `POST /upload/pdf`, `POST /upload/xsd`, `POST /upload/data` em sequência, depois `POST /extract`; `session.jobId` preenchido com o jobId retornado
6. `useSSE(session.jobId)` conecta ao `GET /progress/{jobId}`; `ProgressBar` e `ProgressLabel` atualizam em tempo real com `processingPct` e `processingStep`
7. Após extração concluída: `session.extraction` populado com `ExtractionResult`, router navega para `/campos`
8. Erro fatal (PDF inválido, 500 backend): alert vermelho com mensagem de erro, botão "Tentar novamente" redefine o estado para inicial
9. Erro recuperável (XSD sem campos obrigatórios): warning banner visível, fluxo pode continuar
10. `session store` persistido corretamente após extração bem-sucedida

### API Endpoints
[Source: docs/front-end-spec.md#12-integracao-backend]
```typescript
const API_BASE = import.meta.env.VITE_API_URL ?? ''

POST ${API_BASE}/upload/pdf   // FormData com arquivo PDF
POST ${API_BASE}/upload/xsd   // FormData com arquivo XSD
POST ${API_BASE}/upload/data  // FormData com arquivo JSON/XML
POST ${API_BASE}/extract      // → { jobId: string }
GET  ${API_BASE}/progress/{jobId}  // SSE stream
```
