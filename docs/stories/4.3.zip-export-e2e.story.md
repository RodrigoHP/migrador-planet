# Story 4.3: ZIP Export (FR20) e Validação E2E

## Status

Ready

## Story

**As a** operador da ferramenta,
**I want** baixar o output do template como arquivo ZIP autocontido,
**so that** eu possa entregar o pacote completo (index.html + css/ + js/ + exemplo.js) pronto para uso no motor PDF Template, conforme exigido pelo FR20.

## Acceptance Criteria

1. `GET /api/export/{job_id}/zip` retorna um arquivo ZIP com `Content-Type: application/zip` e `Content-Disposition: attachment; filename="template-{job_id}.zip"`
2. O ZIP contém exatamente: `index.html`, `css/style.css`, `js/base.js`, `exemplo.js`
3. `index.html` contém bindings Knockout.js (`data-bind="text: ..."`) e o placeholder `##TEMPLATE_DATA##` (FR16)
4. Abrindo `index.html` localmente com `exemplo.js` o documento renderiza no browser sem erros (NFR7)
5. Endpoint valida `job_id` como UUID v4 strict — retorna 400 para valores inválidos
6. Endpoint retorna 404 se job não encontrado, 409 se job ainda processando, 422 se job com erro
7. Frontend (`ExportarPage.vue`) chama `GET /api/export/${jobId}/zip` (com prefixo `/api/`)
8. Frontend inicia download automático do ZIP com nome `template-{jobId}.zip`
9. Botão "Download ZIP" fica habilitado apenas quando `generation.previewJobId` está definido e `canDownload` é true
10. Fluxo E2E completo funciona sem erros de console (404, CORS, 422): Upload → Análise → Campos → Layout → Geração → Exportar → Download ZIP

## Dev Notes

### Causa Raiz do Gap G7

O frontend (`ExportarPage.vue`) chama `GET /export/${generation.previewJobId}/zip` mas o backend nunca implementou esse endpoint. Toda a geração de template (HTML, CSS, JS, exemplo) está salva no `job.result` do job — este endpoint apenas empacota esses artefatos em um ZIP.

### Implementação do endpoint ZIP

```python
# backend/routers/export.py
import io
import re
import zipfile
from fastapi import APIRouter, HTTPException
from fastapi.responses import Response
from services import job_manager

router = APIRouter()

_UUID_RE = re.compile(
    r"^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$"
)

def _validate_job_id(job_id: str) -> str:
    if not _UUID_RE.match(job_id.lower()):
        raise HTTPException(status_code=400, detail="jobId inválido: deve ser um UUID v4.")
    # Canonicalization: garante que path resolvido está dentro de TMP_BASE (path traversal prevention)
    TMP_BASE = Path("/tmp/jobs")
    resolved = (TMP_BASE / job_id).resolve()
    if not str(resolved).startswith(str(TMP_BASE.resolve())):
        raise HTTPException(status_code=400, detail="jobId inválido: path fora do escopo permitido.")
    return job_id

@router.get("/export/{job_id}/zip")
async def export_zip(job_id: str):
    """Empacota os artefatos gerados como ZIP autocontido (FR20)."""
    _validate_job_id(job_id)

    job = job_manager.get_job(job_id)
    if job is None:
        raise HTTPException(status_code=404, detail="Job não encontrado")
    if job.status in ("pending", "running"):
        raise HTTPException(status_code=409, detail="Job ainda em processamento")
    if job.status == "error":
        raise HTTPException(status_code=422, detail=job.error_msg or "Erro desconhecido")

    result = job.result or {}
    html = result.get("html", "<!-- template vazio -->")
    css = result.get("css", "")
    js = result.get("js", "")
    exemplo = result.get("exemplo", "")

    # Montar ZIP em memória
    buffer = io.BytesIO()
    with zipfile.ZipFile(buffer, "w", zipfile.ZIP_DEFLATED) as zf:
        zf.writestr("index.html", html)
        zf.writestr("css/style.css", css)
        zf.writestr("js/base.js", js)
        zf.writestr("exemplo.js", exemplo)
    buffer.seek(0)

    return Response(
        content=buffer.read(),
        media_type="application/zip",
        headers={
            "Content-Disposition": f'attachment; filename="template-{job_id}.zip"'
        },
    )
```
