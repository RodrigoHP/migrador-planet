# Story 1.3: Step 2 — Campos

## Status

Draft

## Story

**As a** Operador Técnico,
**I want** revisar o mapeamento de campos gerado pela IA em uma tela com três painéis — PDF viewer, tabela de campos e painel de detalhes — podendo corrigir manualmente campos com confiança baixa ou não encontrados,
**so that** o mapeamento final reflita com precisão a correspondência entre campos XSD e texto do PDF antes de avançar para o layout.

## Acceptance Criteria

1. Rota `/campos` renderiza `CamposPage` com layout de 3 painéis (PDF 50% | Tabela 30% | Detalhes 20%)
2. `PDFViewer` organism exibe o PDF carregado usando PDF.js (lazy import), com navegação de páginas
3. Ao selecionar um campo na tabela, `PDFViewer` destaca o bounding box do campo na página correspondente (`pageRef`)
4. `FieldMappingTable` exibe todos os campos com colunas: Campo XSD, Texto PDF, Tipo, Confiança, Status
5. Cada campo exibe `FieldStatusBadge` (ok/ambiguous/not_found/optional) e `ConfidenceBadge` (high/medium/low)
6. Clicar em um campo abre `FieldDetailPanel` com detalhes, candidatos alternativos e inline editor para `jsonPath`
7. Campo editado manualmente exibe `ManualEditIndicator` (ícone diferenciador); `isManual: true` no store
8. Botão "Confirmar Mapeamento" desabilitado se existir qualquer campo required com status `not_found`
9. Ao confirmar: `mappingStore.confirmed = true`, router navega para `/layout`
10. `SplitPaneLayout` com largura mínima de 280px por painel

## Dev Notes

### Mapping Store — Interface FieldMapping
[Source: docs/front-end-spec.md#9-stores-pinia-e-composables]
```typescript
interface FieldMapping {
  id: string
  pdfText: string
  jsonPath: string
  type: 'text' | 'date' | 'currency' | 'list' | 'composite'
  confidence: 'high' | 'medium' | 'low'
  status: 'ok' | 'ambiguous' | 'not_found' | 'optional'
  candidates?: string[]
  isManual: boolean
  pageRef?: number
  boundingBox?: { x: number; y: number; width: number; height: number }
}
```

### PDF.js Lazy Import
[Source: docs/front-end-spec.md#11-performance]
```typescript
// NUNCA importar no topo do arquivo — deve ser lazy
const pdfjs = await import('pdfjs-dist')
pdfjs.GlobalWorkerOptions.workerSrc = '/pdf.worker.min.js'
```
Copiar `pdfjs-dist/build/pdf.worker.min.js` para `public/` ou configurar via Vite asset.

### Layout 3 painéis
[Source: docs/front-end-spec.md#4-especificacao-de-telas]
- Esquerda: PDF.js preview (50%)
- Centro: tabela de campos (30%)
- Direita: painel de detalhes do campo selecionado (20%)
- Largura mínima por painel: 280px
