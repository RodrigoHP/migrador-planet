# Story 4.9: Hotfix UX — Header Salvar Projeto e WizardStepper 5 Steps

## Status

Ready

## Story

**As a** operador da ferramenta,
**I want** que o header do wizard tenha o botão "Salvar projeto" e que o stepper mostre apenas os 5 steps corretos com acentuação correta,
**so that** eu possa salvar o projeto a qualquer momento durante o wizard e tenha orientação visual clara sobre minha posição no fluxo.

## Acceptance Criteria

1. `AppHeader.vue` exibe botão `[💾 Salvar projeto]` quando `showSave` prop é `true` — ao clicar, chama `useProject().save()`
2. O botão `[💾 Salvar projeto]` aparece no header a partir da Tela 2 (CamposPage) — `UploadPage` passa `:show-save="false"`, as demais passam `:show-save="true"`
3. O botão Salvar tem estado `loading` durante a serialização e exibe feedback de sucesso/erro via `AppToast`
4. `WizardStepper.vue` exibe exatamente 5 steps: `Upload → Campos → Layout → Geração → Exportar` — step "Inicio" removido
5. Labels dos steps corrigidos: `['Upload', 'Campos', 'Layout', 'Geração', 'Exportar']` (com acento em Geração)
6. `currentStep` prop do WizardStepper continua mapeando corretamente: `1=Upload, 2=Campos, 3=Layout, 4=Geração, 5=Exportar`
7. `tsc --noEmit` passa sem erros
8. `npm run build` conclui sem erros

## Dev Notes

### Bug B-UX-H1 — Salvar projeto ausente no header

O wireframe (Componentes Globais) especifica `[💾 Salvar projeto]` no header a partir da Tela 2. Atualmente `AppHeader.vue` só tem o botão Bibliotecas. O botão Salvar deve:
- Usar `useProject().save()` que já existe em `frontend/src/composables/useProject.ts`
- Ter estado loading (desabilitado + spinner) durante a serialização
- Retornar feedback via AppToast (que já está implementado mas não montado — ver Story 4.8)

**Nota:** Se Story 4.8 não tiver sido implementada ainda, integrar AppToast ao WizardLayout como parte desta story.

```typescript
// AppHeader.vue — adição
const props = defineProps<{
  showBibliotecas?: boolean
  showSave?: boolean  // ADICIONAR
}>()

const { save } = useProject()
const isSaving = ref(false)

async function handleSave() {
  isSaving.value = true
  try {
    await save()
    // emit success toast
  } catch {
    // emit error toast
  } finally {
    isSaving.value = false
  }
}
```

### Bug B-UX-H2 — WizardStepper com 6 steps

```typescript
// WizardStepper.vue — ATUAL (errado):
const STEP_LABELS = ['Inicio', 'Upload', 'Campos', 'Layout', 'Geracao', 'Exportar']

// CORRETO:
const STEP_LABELS = ['Upload', 'Campos', 'Layout', 'Geração', 'Exportar']
```

O stepper recebe `currentStep` das pages (1–5). Com 6 labels, o step 0 "Inicio" aparecia como primeiro dot ativo — confusão visual. Com 5 labels, `currentStep=1` ativa o primeiro dot (Upload) corretamente.

### Bug B-UX-H3 — "Geracao" sem acento

Simples correção de string: `'Geracao'` → `'Geração'`.
