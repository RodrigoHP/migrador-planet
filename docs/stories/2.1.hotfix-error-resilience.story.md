# Story 2.1: Hotfix — Error Resilience

## Status

Ready for Review

## Story

**As a** dev agent,
**I want** corrigir os 2 itens HIGH identificados no quality gate arquitetural do Epic 1,
**so that** o frontend seja resiliente a arquivos corrompidos e URLs inválidas antes do deploy em produção.

## Acceptance Criteria

1. `useProject.load()` envolto em try/catch — erros de JSON malformado ou versão incompatível exibem mensagem de erro via `session.setError()` sem crash da aplicação
2. Validação de versão do arquivo de projeto: se `data.version !== '1.0'` lança erro com mensagem "Arquivo de projeto incompatível. Versão esperada: 1.0"
3. Router contém rota catch-all `{ path: '/:pathMatch(.*)*', name: 'not-found', redirect: '/' }` como última rota
4. Navegação para URL inválida redireciona para `/` sem tela em branco
5. `tsc --noEmit` passa sem erros após as mudanças
6. `npm run build` conclui sem erros
