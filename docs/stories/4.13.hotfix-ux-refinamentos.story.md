# Story 4.13: Hotfix UX — Refinamentos: Home Cards, Upload Hints, Layout Desfazer e ExportChecklist

## Status

Ready

## Story

**As a** operador da ferramenta,
**I want** que a tela Home tenha cards explicativos, o Upload dê dicas contextuais sobre os arquivos necessários, o Layout permita desfazer ajustes e o Exportar mostre informações relevantes ao operador (não técnicas),
**so that** o fluxo seja mais guiado, intuitivo e focado no que o operador precisa saber — sem expor detalhes técnicos desnecessários.

## Acceptance Criteria

1. `HomePage.vue` exibe dois cards com descrição textual interna: "Iniciar migração de um novo documento PDF" (card Novo Projeto) e "Retomar projeto salvo (.json)" (card Abrir Projeto)
2. `HomePage.vue` tem botão `[📚 Bibliotecas]` no header — requer que `HomePage` use um layout com `AppHeader` ou implemente o botão diretamente
3. `UploadPage.vue` exibe mensagem de orientação quando nenhum arquivo carregado: "Envie ao menos o PDF + XSD ou PDF + arquivo de dados para continuar"
4. `UploadPage.vue` exibe dica contextual quando PDF+XSD sem dados: "💡 Adicionar dados de exemplo (.xml ou .json) melhora a identificação de formatos (moeda, data, CPF...) e reduz campos com baixa confiança"
5. `UploadPage.vue` exibe dica contextual quando PDF+dados sem XSD: "💡 Adicionar o XSD permite validar se os dados estão completos e identificar campos obrigatórios ausentes antes de prosseguir"
6. `LayoutPage.vue` tem botão `[↩ Desfazer ajustes]` — restaura o layout ao estado hidratado do IDB (estado inicial do step) sem reprocessar
7. `ExportChecklist.vue` exibe itens voltados ao operador, não detalhes técnicos internos — remover itens como "Preview Job ID disponível" e "Referência relativa de bibliotecas" que não têm significado para o operador
8. `tsc --noEmit` passa sem erros
9. `npm run build` conclui sem erros

## Dev Notes

### Bug B-UX-HM1 — HomePage sem cards

```
Wireframe (Tela 0):
┌──────────────────────────────┐   ┌──────────────────────────────┐
│        ➕ Novo Template       │   │       📂 Abrir Projeto       │
│  Iniciar migração de um      │   │  Retomar projeto salvo       │
│  novo documento PDF          │   │  (.json)                     │
│      [ Começar → ]           │   │      [ Carregar arquivo ]    │
└──────────────────────────────┘   └──────────────────────────────┘
```

Implementação atual: dois botões simples sem contexto. Cards ajudam o operador que acessa pela primeira vez.

### Bug B-UX-HM2 — Bibliotecas ausente na Home

O wireframe mostra `[📚 Bibliotecas]` na Home para pré-configuração antes de iniciar o wizard. A HomePage não usa `WizardLayout` nem `AppHeader` atualmente. **Opção A** (adicionar AppHeader) é mais consistente com o design system.

### Bug B-UX-HM3 — Sem mensagem de orientação no Upload

O wireframe especifica a mensagem de requisito mínimo quando nenhum arquivo está carregado. Esta mensagem orienta o operador sobre o mínimo necessário para avançar.

### Bug B-UX-HM4 — ExportChecklist técnica demais

```
Wireframe (anotações Tela 5):
"Validação técnica é responsabilidade do sistema, não do operador.
Esta tela é um resumo de conclusão — o operador chegou aqui porque tudo funcionou."
```

A ExportChecklist atual mostra "Preview Job ID disponível" e "Referência relativa de bibliotecas" — detalhes técnicos sem valor para o operador. A simplificação melhora a experiência sem comprometer a funcionalidade (o botão Download ainda usa a lógica de validação internamente se necessário).

### Bug B-UX-HM5 — Hints contextuais do Upload

```
Wireframe estados Upload:

Estado: PDF + dados (sem XSD)
💡 "Adicionar o XSD permite validar se os dados estão completos..."

Estado: PDF + XSD (sem dados)
💡 "Adicionar dados de exemplo (.xml ou .json) melhora..."
```

Estas dicas já existem textualmente no wireframe. Implementação é condicional simples com computed ou v-if nos refs de seleção de arquivo.

### Bug B-UX-L1 — Desfazer ajustes ausente no Layout

O snapshot `originalLayout` deve ser capturado **após** o `hydrateFromIdb()` — representa o estado que o layout tinha ao entrar na tela. O `$patch` com o snapshot restaura todos os valores editáveis de volta ao inicial.
